# Деплой изменений на GitHub Pages

Этот skill помогает задеплоить изменения на GitHub Pages.

## Что нужно сделать:

1. **Собрать сайт** (если еще не сделано):
```bash
npx eleventy
```

2. **Проверить статус git**:
```bash
git status
```

3. **Добавить изменения в staging**:
```bash
git add .
```

4. **Создать коммит** с понятным сообщением:
```bash
git commit -m "Описание изменений"
```

Примеры сообщений коммита:
- "Add new event: название события"
- "Add new post: название публикации"
- "Update links: добавлена ссылка на X"
- "Update content on main page"

5. **Запушить изменения** на удаленный репозиторий:
```bash
git push -u origin имя-ветки
```

6. **Проверить деплой**:
   - GitHub Pages автоматически обновит сайт после push
   - Обычно это занимает 1-3 минуты
   - Сайт доступен по адресу: https://matyushkin.github.io/

## Примечания:

- Перед деплоем всегда запускайте сборку через Eleventy
- Убедитесь, что все изменения корректны и проверены
- В коммите указывайте понятное описание изменений
- После push изменения автоматически появятся на сайте через несколько минут
- Проверьте сайт в браузере после деплоя

## Структура веток:

- `main` или `master` - основная ветка для деплоя на GitHub Pages
- Можно создавать feature-ветки для разработки новых функций

## Важно:

- Всегда собирайте сайт перед коммитом
- Проверяйте изменения локально перед push
- Не коммитьте файлы с ошибками или незавершенные изменения
- Регулярно делайте push, чтобы не потерять изменения
